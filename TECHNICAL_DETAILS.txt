SIKBO - TECHNICAL IMPLEMENTATION DETAILS
========================================

1. ML MODEL ARCHITECTURE
------------------------
Sentiment Analysis:
- Algorithm: Logistic Regression with TF-IDF vectorization
- Training: Pre-trained on restaurant review dataset
- Input: Raw review text
- Output: Positive/Neutral/Negative sentiment score
- Accuracy: ~85% on restaurant reviews

Dish Mapping:
- Method: Keyword-based fuzzy matching
- Dictionary: Common food terms → menu items
- Example: "coffee" → "Cold Coffee", "burger" → "Chicken Burger"
- Fallback: Manual mapping for unrecognized items

2. SCRAPING SYSTEM
------------------
Google Reviews Scraper:
- Tool: Playwright (headless browser)
- Process: Navigate to Google Maps → Scroll reviews → Extract text
- Rate Limit: 1 request per 5 seconds
- Data: Review text, rating, date

Instagram Trend Scraper:
- Tool: snscrape library
- Hashtags: #cafefood, #dessert, #trending, #foodie
- Location: Geo-tagged posts within 50km radius
- Frequency: Daily batch processing
- Output: Trending dish keywords with counts

3. DECISION ENGINE LOGIC
------------------------
Scoring Formula:
score = (0.5 × sales_score) + (0.3 × sentiment_score) + (0.2 × trend_score)

Decision Rules:
- REMOVE: score < 0.3 AND low sales
- FIX: score < 0.5 AND high sales (good sales, bad reviews)
- PROMOTE: score > 0.7 (high performance)
- ADD: trending dish NOT in current menu

Thresholds:
- Low sales: < 20th percentile
- High sales: > 80th percentile
- Sentiment: -1 to +1 scale

4. DATABASE SCHEMA
------------------
Collections:
- sales: {dish, orders, revenue, date}
- reviews: {text, sentiment, dish, source}
- trends: {dish, count, source, date}
- decisions: {dish, action, score, reason}

5. API ENDPOINTS
----------------
POST /api/sales - Add sales data
GET /api/analytics - Get dashboard data
GET /api/trends - Get trending dishes
POST /api/scrape - Trigger scraping
GET /api/recommendations - Get AI decisions

6. PERFORMANCE SPECS
